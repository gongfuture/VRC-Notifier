[功能说明]

本工具是后台自动监控工具，用于在后台静默监控您的好友状态变化，并通过邮件和平台通知您。

  • 本工具不会在前台实时显示好友状态（如有此需求，请使用 VRCX）
  • 开启监控后，程序会在后台自动运行，当好友上线/下线/切换世界时发送通知
  • 您无需保持网页打开，只需保持后端程序运行即可
  • 建议将本程序部署在服务器或长期运行的设备上

[API] 请求逻辑说明

  • 登录时：仅获取在线和离线好友列表（2次API请求），不获取世界信息
  • 手动刷新时：获取好友列表信息，刷新后60秒内暂停监控
  • 监控检查时：每30秒检查一次，仅获取被监控好友的世界信息（轮询机制，每10秒一个好友）
  • 冷却期：登录或刷新后60秒内暂停所有监控，防止API冲突

[安全] API保护机制

本工具内置多重API保护机制，确保您的账号安全：

  • 请求频率限制：严格遵循VRChat API规范，用户资料≤1次/分钟，好友状态≤2次/分钟，世界信息≤6次/分钟
  • 自动限流保护：当检测到请求频率过高时，系统自动暂停60秒，防止触发官方限制
  • 实时弹窗提醒：触发限流保护时，会显示红色弹窗提醒用户
  • 保守策略：所有API请求频率均低于官方限制，留有充足安全余量
  • 只要正常使用，不疯狂调用API，不会出现账号封禁问题

[!] 使用提示

  • 在标准模式下，最多可监控5个好友，只有被监控的好友才会获取世界信息
  • 如遇问题，可使用"退出并清除数据"按钮清除所有本地数据后重新登录
  • 请勿频繁刷新好友列表，避免触发API限流
  • 与VRCX同时使用时可能产生API冲突，但本工具已预留大量冗余，不会影响VRCX正常使用

[X] 退出并清除数据

点击"退出并清除数据"按钮将：

  • 登出当前账号
  • 提示您手动删除 data 文件夹以清除所有数据

适用于：数据异常、缓存问题、需要重置所有设置时

[?] 什么是"仅监控状态"模式？

这是一个简化版监控模式，只关注好友的在线/离线状态，不关注他们在哪个世界。

[+] 为什么要使用这个模式？

  • 好友数量多：标准模式最多只能监控5个好友，此模式可以监控所有好友
  • 节省API配额：不查询世界信息，API调用从8次/分钟降至2次/分钟
  • 简化通知：只关心好友是否在线和状态，不关心他们在哪个房间
  • 降低服务器负载：减少API请求，对服务器更友好

[↻] 两种模式的区别

功能 标准模式 仅状态模式
监控数量 最多5个 无限制
在线状态 支持 支持
世界变化 支持 不支持
API调用 8次/分钟 2次/分钟

[!] 如何使用？

  1. 点击"仅监控状态"开关开启此模式
  2. 确认切换（会自动关闭当前所有监控的好友）
  3. 点击"一键关闭监控全部好友"按钮关闭所有好友
  4. 完成！您将收到所有好友的上线/下线通知

[⚠] 注意事项

  • 切换模式时会自动关闭所有监控的好友，需要重新选择
  • 此模式下不会收到世界变化通知（如好友切换房间）
  • 如果想监控特定好友的世界变化，请使用标准模式
  • 两种模式可以随时切换，但每次切换都会重置监控列表

使用说明

什么是 Gotify?

Gotify 是一个开源的自托管消息推送服务器，可以将通知推送到您的 Android 设备或浏览器。 您需要在服务器上部署 Gotify，然后在手机上下载 Gotify App 来接收推送。

配置步骤:
  1. 在您的服务器上部署 Gotify（Docker 部署最简单）
  2. 登录 Gotify Web 界面，创建一个新的 Application
  3. 复制生成的 Token，填写到上面的"应用 Token"字段
  4. 填写您的 Gotify 服务器地址
  5. 点击"发送测试推送"验证配置
  6. 在手机上下载 Gotify App，登录后即可接收推送
Docker 快速部署 Gotify: docker run -p 8080:80 -v /var/gotify/data:/app/data gotify/server

使用说明

什么是通用 Webhook?

通用 Webhook 允许您将通知推送到任何支持 HTTP 请求的第三方服务，如 Discord、Slack、企业微信、钉钉、飞书等。 只需配置目标 URL 和请求格式，即可实现消息推送。

支持的推送平台示例:
  • Discord - 使用 Discord Webhook URL
  • Slack - 使用 Slack Incoming Webhook
  • 企业微信 - 使用企业微信群机器人 Webhook
  • 钉钉 - 使用钉钉群机器人 Webhook
  • 飞书 - 使用飞书群机器人 Webhook
  • 自定义服务 - 任何支持 HTTP 的接口

📡 API 请求逻辑说明

1. 登录时

  • 获取在线好友列表（1次API请求）
  • 获取离线好友列表（1次API请求）
  • 不获取任何世界信息

2. 手动刷新好友列表时

  • 获取在线好友列表（1次API请求）
  • 获取离线好友列表（1次API请求）
  • 获取世界信息（仅前10个好友）
  • 刷新后60秒内暂停监控

3. 监控检查时（每30秒）

  • 获取在线好友列表（1次API请求）
  • 仅获取被监控好友的世界信息
  • 轮询机制：每10秒检查一个被监控好友

4. 冷却期机制

  • 登录后60秒内暂停监控
  • 手动刷新后60秒内暂停监控
  • 防止API请求冲突和限流

[!] 使用提示

  • 最多可监控5个好友
  • 只有被监控的好友才会获取世界信息
  • 如遇问题，可使用"退出并清除数据"按钮重置
  • 请勿频繁刷新好友列表，避免触发API限流
